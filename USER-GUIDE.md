# AI Proxy 使用指南

> 面向企业员工的 AI 工具接入手册

---

## 为什么上线 AI Proxy？

公司为大家统一搭建了 AI 模型服务平台（AI Proxy），相比各自注册账号使用，它有以下优势：

- **飞书一键登录** — 无需单独注册，用飞书扫码即可登录，自动关联公司身份
- **用量透明可查** — 随时查看自己的使用量、消耗额度、请求次数，心中有数
- **模型齐全统一** — 180+ 主流模型（GPT、Claude、DeepSeek、Gemini、Kimi 等）一站式接入，无需分别注册各家服务
- **成本公司承担** — 使用额度由公司统一结算，个人无需付费

### 你能用到的三个板块

登录后左侧导航栏有三个板块，覆盖了日常使用的全部需求：

| 板块 | 一句话说明 |
|------|-----------|
| **📋 我的接入** | 获取 Key、配置工具、查看可用模型 — **新用户从这里开始** |
| **📊 企业概览** | 全公司的使用数据总览、部门消耗排名、模型热度 |
| **🏆 员工排行** | 个人使用量排行榜，看看谁是 AI 达人 |

> 💡 **居家办公提示：** 日常使用 AI 模型（通过 Cherry Studio、Cursor、Claude Code 等工具调用 API）**无需开启 VPN**，直接使用即可。但如果需要登录前端管理系统（`https://ai.paigod.work`）查看统计数据或管理 Key，请先接入公司 VPN。VPN 配置指南：[点击查看飞书文档](https://ppio-cloud.feishu.cn/wiki/Mwj2wNdjRizvkAkmEeqctwrBnDh?from=from_copylink)

### 🤖 Claude Code

> Anthropic 官方命令行 AI 编程助手，适合代码开发和自动化任务。

**配置方式：** 通过环境变量设置

**macOS / Linux：**

在终端中执行：

```bash
# 设置环境变量（添加到 ~/.bashrc 或 ~/.zshrc 中，永久生效）
export ANTHROPIC_BASE_URL=https://apiproxy.paigod.work
export ANTHROPIC_API_KEY=sk-xxxxxxxxxxxx
```

或者在启动时指定：

```bash
ANTHROPIC_BASE_URL=https://apiproxy.paigod.work \
ANTHROPIC_API_KEY=sk-xxxxxxxxxxxx \
claude
```

**Windows（PowerShell）：**

```powershell
$env:ANTHROPIC_BASE_URL = "https://apiproxy.paigod.work"
$env:ANTHROPIC_API_KEY = "sk-xxxxxxxxxxxx"
claude
```

**操作步骤：**

1. 确保已安装 Claude Code（`npm install -g @anthropic-ai/claude-code`）
2. 设置上面的两个环境变量
3. 在终端输入 `claude` 启动
4. 如果提示选择模型，推荐选择 `pa/claude-sonnet-4-6`（最新版 Claude Sonnet）

> 💡 **注意**：Claude Code 使用 Anthropic 协议，Base URL **不带** `/v1`（与 OpenAI 兼容工具不同）。
>
> 💡 **建议**把环境变量写入 `~/.zshrc`（macOS）或 `~/.bashrc`（Linux），避免每次手动输入。

## 三、协议与端点说明

### 什么是"端点"？

**端点**就是 AI 工具和我们服务器"对话"的方式。不同的 AI 工具使用不同的"语言"（协议），我们的平台全部支持，并且能自动互相转换。

**你不需要关心端点细节** — 只要按照上面的配置步骤填好 Base URL 和 API Key，工具会自动选择正确的端点。

### 三种协议，一句话理解

| 协议 | 一句话说明 | Base URL | 哪些工具在用 |
|------|-----------|----------|-------------|
| **OpenAI 兼容** | 最通用的格式，90% 的工具都支持 | `https://apiproxy.paigod.work/v1` | Cherry Studio、Cursor、ChatBox、LobeChat 等 |
| **Anthropic 兼容** | Claude 系列的原生格式 | `https://apiproxy.paigod.work`（不带 `/v1`） | Claude Code |
| **MCP 协议** | AI Agent 工具的专用协议 | `https://apiproxy.paigod.work/mcp` | 支持 MCP 的 Agent 工具 |

> 💡 **不确定选哪个？** 选 **OpenAI 兼容**，在工具里选 "OpenAI" 或 "自定义 API"，填入 Base URL 和 API Key 即可。

### 模型列表中的端点标签是什么意思？

在「我的接入」页面的模型列表中，每个模型下方会显示彩色端点标签，表示该模型支持哪些调用方式：

| 端点标签 | 含义 | 说明 |
|---------|------|------|
| **Chat** | OpenAI Chat 格式 | 最常用的对话格式，绝大多数工具都用这个 |
| **Completions** | OpenAI 补全格式 | 旧版文本补全格式，少数工具会用 |
| **Anthropic** | Claude Messages 格式 | Claude Code 等 Anthropic 原生工具使用 |
| **Responses** | OpenAI Responses 格式 | 部分新版 GPT 模型的专用格式 |
| **Embeddings** | 文本向量化 | 用于语义搜索、知识库构建，非对话用途 |
| **Rerank** | 结果重排序 | 搜索结果优化，非对话用途 |
| **Web Search** | 实时网页搜索 | 独立的搜索接口，可被 AI 工具作为 tool/skill 调用 |

**常见组合解读：**

- **Chat + Completions + Anthropic**（3个标签）— 标准对话模型，所有主流工具都能用
- **Chat + Completions + Anthropic + Responses**（4个标签）— 功能最全的模型，额外支持 Responses 格式
- **Responses**（仅1个标签）— Codex 编程专用模型（如 `pa/gpt-5.3-codex`），仅支持 Responses 格式，需要用支持该格式的工具
- **Embeddings**（仅1个标签）— 向量模型，用于知识库/搜索场景，不能用来聊天

> 💡 **日常使用只需关注有 "Chat" 标签的模型**，这些模型在 Cherry Studio、Cursor 等工具中都能直接使用。

### 接口信息

| 项目 | 值 |
|------|---|
| **端点** | `POST https://apiproxy.paigod.work/v1/web-search` |
| **虚拟模型** | `ppio-web-search`（请求体中必须包含此字段） |
| **认证** | 与其他接口相同，`Authorization: Bearer sk-xxxx` |

### 请求示例

```bash
curl -X POST https://apiproxy.paigod.work/v1/web-search \
  -H "Authorization: Bearer sk-xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "ppio-web-search",
    "query": "React 19.1 新特性",
    "count": 5
  }'
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model` | string | 是 | 固定填 `ppio-web-search` |
| `query` | string | 是 | 搜索关键词 |
| `count` | int | 否 | 返回结果数量（默认 10，最大 50） |
| `freshness` | string | 否 | 时效过滤：`24h`、`7d`、`30d` |
| `summary` | bool | 否 | 是否返回摘要（默认 false） |

### 响应格式

返回标准搜索结果结构（Bing Search 兼容格式），包含 `webPages`、`images`、`videos` 等字段。

## 六、常见问题

### 连接不上 / 报错 401

- 检查 API Key 是否正确复制（前后不要有空格）
- 确认 Base URL 是 `https://apiproxy.paigod.work/v1`（OpenAI 兼容工具）或 `https://apiproxy.paigod.work`（Anthropic 工具，不带 `/v1`）
- 注意 **不是** `ai.paigod.work`（那是内网管理后台地址）
- API Key 过期？重新登录 `https://ai.paigod.work` 获取新 Key

### 提示模型不存在

- 确认模型 ID 拼写正确（区分大小写）
- 闭源模型需要带 `pa/` 前缀（如 `pa/gpt-5.4` 而非 `gpt-5.4`）
- 开源模型使用完整路径（如 `deepseek/deepseek-r1-0528`）
- 部分模型可能未开放，联系管理员确认

### 响应很慢

- AI 模型首次响应需要 2-5 秒属于正常（这叫"首字延迟"）
- 复杂问题 + 长上下文会更慢
- 推理模型（如 DeepSeek-R1）会先"思考"再回答，等待时间更长是正常的
- 尝试换用更快的模型（如 Claude Haiku 4.5 或 DeepSeek-V3 Turbo）

### 提示 "Responses" 格式不支持

- 部分 Codex 编程模型（如 `pa/gpt-5.3-codex`）仅支持 Responses 格式
- 这些模型需要使用支持 Responses API 的工具（如 OpenAI 官方 SDK）
- 如果你的工具不支持，换用对应的非 Codex 版本即可（如用 `pa/gpt-5.4` 替代）

### Claude Code 报 504 / "Cannot read properties of undefined (reading 'trim')"

> 这是同一个问题的两种表现，通常在使用 Claude Code 时遇到。

**典型错误信息：**

```
API Error: 504 <html><head><title>504 Gateway Time-out</title></head>...

API Error: Cannot read properties of undefined (reading 'trim')
```

**原因：** Claude Code 使用 extended thinking（深度思考）时，单次请求可能持续 3-10 分钟。如果网络中间层（反向代理或负载均衡）的超时设置过短，会在 AI 还没回复完时就断开连接，返回一个 HTML 格式的 504 错误页面。Claude Code 期望收到 JSON 格式的响应，解析 HTML 失败就会报第二个错误。

**解决方案：** 这是服务端配置问题，**请联系管理员**，让管理员参照部署文档 §3.6 和 §7.1 将 Nginx 和 CLB 超时调整到 900 秒。

**临时缓解：** 如果管理员暂时无法修改，可以尝试：
- 减少单次对话的上下文长度（减少 token 数），缩短 AI 思考时间
- 使用较轻量的模型（如 `pa/claude-haiku-4-5-20251001`），响应更快

### 提示 429 "request tpm/rpm limit exceeded"

> 请求频率或 Token 消耗速度超过了平台设定的限制。

**典型错误信息：**

```
429 {"type":"error","error":{"type":"aiproxy_error","message":"request tpm limit exceeded, please try again later"}}
```

**含义说明：**

| 错误信息 | 含义 |
|---------|------|
| `request **rpm** limit exceeded` | 每分钟请求次数超限（发送太频繁） |
| `request **tpm** limit exceeded` | 每分钟 Token 消耗超限（单次或累计输入/输出 Token 太多） |

**解决方法：**

- **等一会再试** — 限制窗口为 1 分钟，稍等片刻后即可恢复
- **减少请求频率** — 避免短时间内连续发送大量请求
- **缩短对话上下文** — 每次请求携带的历史消息越多，Token 消耗越大。尝试新建对话或减少上下文长度
- **换用更轻量的模型** — 不同模型的限流阈值可能不同
- **联系管理员** — 如果经常触发限流，可以请管理员调整你所用模型的 RPM/TPM 上限

> 💡 **TPM 和 RPM 是什么？**
> - **RPM（Requests Per Minute）**：每分钟最多发多少次请求，不管每次请求多大
> - **TPM（Tokens Per Minute）**：每分钟最多消耗多少 Token（输入+输出合计）
>
> 两者互相独立，任意一个超限都会触发 429。

### 配额用完了怎么办？

如果管理员为你设置了使用配额，当消耗接近或超过配额时：

- **接近配额**（如用了 70%-90%）：系统可能会降低你的请求速度限制（RPM/TPM），但仍可使用
- **达到配额**（100%）：可能无法继续使用，需等待下个周期重置或联系管理员提升额度
- **查看配额状态**：在「我的接入」页面的「配额状态」卡片中查看当前周期的额度、已用量、使用比例

> 💡 配额按周期（通常为自然月）自动重置，无需手动操作。

### 其他 AI 工具怎么配？

大部分 AI 工具都支持 **OpenAI 兼容** 模式，通用配置方法：

1. 在工具设置中找到 "API" 或 "模型提供商" 相关选项
2. 选择 "OpenAI" 或 "自定义 API"
3. 填入 Base URL：`https://apiproxy.paigod.work/v1`
4. 填入 API Key：`sk-xxxx`
5. 保存并测试
